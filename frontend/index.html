⚡ Problem statement

File sharing is still weirdly painful in real life

In classrooms, labs, or group work, people often need to move a file quickly from one device to another. The usual options are slow or awkward.

🧨 What goes wrong
  • Typing links on mobile is error-prone and wastes time.
  • Chat apps compress files, clutter groups, and lose important context.
  • “Drive-style” sharing needs accounts/permissions and isn’t instant.
  • USB/Bluetooth is inconsistent across devices and setups.
🎯 What this app solves
  • One upload creates a single shareable QR + link.
  • One scan opens a clean download page—no typing.
  • Controlled access through the server (expiry + tracking).
  • Works anywhere with a browser—no special app needed.
Goal: make short‑term, in‑person file handoffs as fast as showing your screen.
✦ About this website

Why “Upload → QR → Scan → Download” works

This flow is built for quick, in-person sharing (classrooms, labs, group work) where typing links is slow and error-prone.

✅ Benefits
  • Fast: share a file in seconds—scan beats typing.
  • Works across devices: any phone camera can open the download page.
  • Safer than “public links”: downloads go through the server so we can enforce rules.
  • Trackable: see how many times a file was downloaded.
  • Clean UX: one QR per upload, one page per file—no confusion.
⚠️ Drawbacks (and how we handle them)
  • Links can be shared further once someone has it → files expire automatically.
  • Storage cost for uploaded files → TTL cleanup reduces wasted storage.
  • Need internet to upload/download → optimized for real-time sharing, not offline transfer.
  • Access control matters for uploads → login is required to upload.
  • Large files take time → clear limits + progress messaging.
Best for: notes, PDFs, assignments, images, and quick group handoffs. Not ideal for long-term archival.
